Overview

A 3D chase game, built from scratch in the browser

Road Rogue is a 3D car chase game inspired by Smashy Road — a mobile game where you drive a getaway car through a city while police vehicles pursue you, with the goal of surviving as long as possible. The challenge was to recreate that core tension in a browser environment using Three.js and WebGL.

Building a real-time 3D game without a game engine is a significant technical challenge: collision detection, physics-like movement, camera systems, asset loading, and performance optimization all have to be built from first principles. Road Rogue was a project for pushing past tutorial territory and into genuine problem-solving.

Technical Work

Building game systems from scratch with Three.js

Three.js provides the 3D rendering layer — managing scenes, cameras, lights, and meshes — but game logic is a separate problem that had to be built independently. The key systems built for Road Rogue:

  • Driving controls — keyboard input handling with acceleration, braking, and steering that feels responsive and satisfying. Car movement uses a simple physics model with velocity and friction, rather than just direct position updates, to give the driving a sense of weight.
  • Procedural city layout — the city is assembled from modular low-poly building and road blocks, randomly arranged at runtime so each playthrough feels different.
  • Police AI — pursuit vehicles use basic steering behavior to follow the player, with awareness of the city grid to avoid getting stuck. Multiple police cars are spawned progressively as the score increases, ramping up pressure over time.
  • Collision detection — AABB (Axis-Aligned Bounding Box) collision between the player car, police cars, and buildings. On a collision with a building, speed drops; on collision with a police car, the game ends.
  • Score and difficulty scaling — a survival timer drives the score, and police spawn rate and speed increase as the score climbs, creating escalating tension.
  • Camera system — a third-person follow camera that stays behind the player car, smoothly interpolating position and rotation for a cinematic feel while keeping the city visible ahead.

Low-poly 3D assets — buildings, the player car, police vehicles — were modeled in Blender, exported as GLTF, and loaded into Three.js at runtime. Keeping geometry simple was both an aesthetic choice and a performance necessity for smooth browser rendering.

Reflection

What building a game teaches you about programming

Games are a uniquely demanding programming environment — the game loop runs 60 times per second, every system interacts with every other system, and the user notices immediately when anything feels wrong. Building Road Rogue forced a level of rigour in performance profiling, state management, and system design that typical web projects don't require.

The project also deepened experience with 3D math: transformation matrices, quaternions for rotation, vector arithmetic for movement and AI steering. Three.js abstracts some of this, but building the game logic still requires understanding what's happening geometrically.

Most importantly: games are fun to build. The feedback loop of making a car drive correctly, then having it chased by police cars, then watching the city come together — is immediate and satisfying in a way that motivates pushing further.
