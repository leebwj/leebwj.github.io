Overview

A portfolio that is itself a project

Art of the Web is a course about the internet as a creative and cultural medium — exploring how the web is built, who controls it, and how it shapes and reflects culture. Rather than submitting work through a learning management system, the format required building and maintaining a public portfolio site throughout the semester.

This site became more than a submission container. It was a design challenge in its own right: how do you create a consistent, well-designed archive of work that is itself exploring questions of web design, internet aesthetics, and interactive media?

What Was Built

Consistent design across a full semester of work

Built entirely in vanilla HTML, CSS, and JavaScript — no frameworks, no libraries — the portfolio site archives every project and reading response from the semester. Each entry has its own page, styled to fit within the site's overall visual system while allowing the content to breathe.

Key design decisions throughout the build:

  • Consistent layout system — a shared grid and type scale applied across all pages, so the archive feels cohesive even as individual projects vary in format.
  • Deliberate interaction design — hover states, transitions, and navigation behaviors were all designed intentionally, not left as browser defaults.
  • Information architecture — the portfolio is organized to be browsable, with clear categories and navigation that makes it easy to move between projects, reading responses, and experiments.
  • Internet aesthetics as content — several projects on the site directly engage with the visual language of the web itself: its layouts, typographies, and interactive conventions.

Working without frameworks reinforced a deep understanding of how CSS and JavaScript actually work — not just how to use libraries that abstract them away.

Reflection

Growth in both craft and thinking

This project represents a semester of consistent, iterative web work — returning to the same codebase week after week and making it better. The discipline of maintaining a public archive of your own learning, while designing that archive to be well-crafted, pushes you to care about the quality of your work at every level.

The course itself — exploring the web as cultural infrastructure — shaped how I think about interface design. Every design choice on the web is a political and cultural act as much as a functional one. That perspective has carried forward into everything since.
